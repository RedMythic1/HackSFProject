Introduction

The Model Context Protocol (MCP) is an open protocol that aims to standardize how applications provide context to LLMs. However, a critical look at the protocol reveals that it is poorly engineered, with subpar documentation and implementation guidance. The transport protocol, based on Server-Sent Events (SSE) and Streamable HTTP, is also criticized, with the author suggesting it should be replaced with something that mimics stdio. This article provides an example of implementing an MCP server in Golang, but found the documentation and implementation to be poor. Ultimately, the author concludes that the industry should optimize for the most common use-cases and not the corner-cases, and that WebSockets are the appropriate choice for transport over HTTP. This article is important for anyone interested in understanding the limitations and potential improvements of the MCP protocol, as well as the broader implications of standardizing context in LLMs.
